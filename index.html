What is HouseBite?

A progressive web app that turns your household into a shared smart pantry — less waste, less spending, less stress.

You join or create a household, and HouseBite becomes your shared food manager. Add products with expiry dates, see what's about to go, and get recipe suggestions that use exactly what you already have at home.

Beyond the pantry, it tracks grocery spending against a monthly budget, lets members scan receipts with on-device OCR to log purchases automatically, and handles dietary restrictions per person — so no one is left out of the meal plan.

30–40% of food bought in Europe is wasted at household level
€600 average annual cost of food waste per European household

Sprint overview

Sprint 1 Completed

Foundation

Project scaffolding, authentication, and the shared household model that everything else sits on top of.

  • Project setup — Vite + React + TypeScript, Docker dev environment
  • Supabase Auth with automatic family_member trigger on sign-up
  • Households: create, join via invite code, member allocations
  • Database schema across 12 tables with Row Level Security
Sprint 2 In progress

Core experience

Building out the features that make HouseBite useful day-to-day — what lands in the hands of the user.

  • Household dashboard with filterable “expiring soon” view
  • Recipe suggestions via Spoonacular — diet and intolerance aware
  • Receipt scanning with on-device OCR (Tesseract.js)
  • Account page: dietary restriction profiles per member

Under the hood

Tech & architecture

HouseBite is a progressive web app — installable on phones and desktops, with a Supabase backend and on-device processing where it matters.

Frontend

  • React 19 + TypeScript
  • Vite + React Compiler
  • Mantine UI + Tabler Icons
  • React Router 7
  • PWA manifest + icons

Backend & data

  • Supabase Auth
  • PostgreSQL with Row Level Security
  • Edge Functions (Deno)
  • SECURITY DEFINER RPCs
  • 12 tables, 1 derived view

Integrations

  • Spoonacular — recipe search
  • Tesseract.js — on-device OCR
  • Open Food Facts — barcode data

Workflow

  • Scrum methodology
  • Sprint-based delivery
  • KTH GitLab + GitHub dual remote
  • Docker for local dev
